# Live TV UX v2 Implementation Plan

> **For agentic workers:** Executed **subagent-driven** — one scoped subagent per Phase, with the
> author validating (build + emulator) between phases. Repo has no unit-test harness except the
> one added in Phase 2 for `RegionClassifier`; everything else is validated by
> `./gradlew assembleDebug` + on-device. Steps use checkbox (`- [ ]`).

**Goal:** Make Live TV cache-first (instant opens), default to UK+US+EN channels via a
Region/Language → Category model, with global channel search, favourite channels & categories, and
recently-watched.

**Architecture:** Room becomes the on-disk source of truth; the repository exposes Flows and
refreshes from the Xtream API in the background (stale-while-revalidate). A pure `RegionClassifier`
derives a region tag from each category name. The ViewModel combines Room Flows into a "For You"
landing; pickers replace the chip strip.

**Tech Stack:** Kotlin, Jetpack Compose, **Room (KSP)**, Hilt, Coil 2.7.0, Retrofit/kotlinx,
DataStore, Media3.

## Global Constraints

- Build: `JAVA_HOME="/Applications/Android Studio.app/Contents/jbr/Contents/Home"`, `ANDROID_HOME="$HOME/Library/Android/sdk"`, `./gradlew assembleDebug`. Emulator `emulator-5554` (provider reachable only via VPN — Virgin blocks it).
- KSP is already configured; add Room via KSP (not kapt). Kotlin 2.0.0 / KSP 2.0.0-1.0.24.
- Do NOT break VOD/search/Detail/player. The Xtream cred resolution (`XtreamCreds.fromGetPhp` + fields) and `.ts` live-stream URLs stay.
- Cache-first: UI always reads Room; network only refreshes Room. Never block the UI on network.
- Default regions: **`{US, UK, EN}`**, persisted in DataStore key `live_regions`.
- Region detection is best-effort from category text; unmatched → `OTHER`; global channel search ignores region so nothing is unreachable.
- Spec: `docs/superpowers/specs/2026-06-21-live-tv-ux-v2-design.md` — read it before each phase.

## Phase 1 — Room cache-first persistence

**Outcome:** the existing Live TV screen renders from Room and persists across cold starts; a
second launch is instant (no re-download). No UI/behaviour change yet beyond persistence.

### Task 1.1: Add Room dependencies
- [ ] In `gradle/libs.versions.toml` add `room = "2.6.1"`; libraries `room-runtime`, `room-ktx` (group `androidx.room`), and `room-compiler` (same version).
- [ ] In `app/build.gradle.kts` add `implementation(libs.room.runtime)`, `implementation(libs.room.ktx)`, `ksp(libs.room.compiler)`.
- [ ] Build: `./gradlew assembleDebug` → BUILD SUCCESSFUL (deps resolve).
- [ ] Commit: `feat(livetv): add Room dependencies`.

### Task 1.2: Entities + DAOs + AppDatabase
- [ ] Create the 7 entities (fields per spec §5.1). Keys: channel `id` (String) PK; category `id` (String) PK; epg `channelId` PK; favourites by id PK; recents `channelId` PK; cacheMeta `key` PK. Index `ChannelEntity.regionTag`, `categoryId`, `name`.
- [ ] Create DAOs (spec §5.2) — reads return `Flow`; include `@Upsert`/`@Query` for: channels observe(regionTags, categoryId?), search(q) global, byId; categories observe(regionTags)/all; favourites observe+toggle+isFav; recents observe(limit)+upsert+prune; epg byId+upsert; meta get+upsert. Use `List<String>` region args with `IN (:regions)`.
- [ ] Create `AppDatabase` (abstract, `@Database(entities=[...], version=1)`, dao getters).
- [ ] Build → SUCCESS (Room codegen compiles). Commit: `feat(livetv): Room entities, DAOs, database`.

### Task 1.3: DI + LiveTvRepository (cache-first)
- [ ] `DatabaseModule` provides `AppDatabase` (`Room.databaseBuilder(...,"aiotv.db").fallbackToDestructiveMigration().build()`) + each DAO.
- [ ] `LiveTvRepository` (Singleton): expose DAO Flows; `refresh(force)` — if forced or stale (cacheMeta `channels`/`categories` older than 12 h) fetch `get_live_streams`/`get_live_categories` (reuse the Xtream cred resolution + filtering + `.ts` URL from `IptvRepository.fetchXtream`), map to entities (regionTag = `""` for now — Phase 2 fills it), upsert, set cacheMeta. `epgFor(channelId,...)` — Room epg if `now < expiresAtMs` else `get_short_epg` → upsert. `clearCache()` wipes tables. All on `Dispatchers.IO`, wrapped so failures don't crash.
- [ ] Build → SUCCESS. Commit: `feat(livetv): cache-first LiveTvRepository over Room`.

### Task 1.4: Point the current screen at Room
- [ ] Update `LiveTvViewModel` to read channels/categories from `LiveTvRepository` Flows (instead of in-memory `IptvRepository`), and call `refresh()` on init (background). EPG via `repository.epgFor`. Keep current UI (categories+list+search) working.
- [ ] Build → SUCCESS.
- [ ] **Author validates on emulator:** open Live TV (loads, persists); force-stop + reopen → **instant from cache** (no get_live_streams on 2nd open within TTL — confirm via logcat). Commit: `feat(livetv): ViewModel reads cache-first Room`.

## Phase 2 — Region/Language classifier + filter (default UK+US+EN)

**Outcome:** channels/categories carry a region tag; Live TV defaults to UK+US+EN; a region
multi-select filters the list. Includes a unit test for the classifier.

### Task 2.1: RegionClassifier + unit test
- [ ] Create `RegionClassifier` with `fun classify(name: String): String` returning one of
  `US, UK, EN, LATAM, EU, MENA, OTHER` per the spec §6 keyword table (upper-case token match,
  ordered so "LATIN AMERICA"→LATAM beats "AMERICA"→US; "USA"/"US"→US; "UK"/"GB"/"BRITISH"/"ENGLAND"→UK;
  "ENGLISH"→EN; country codes DO/EC/HN/HT/PA/MX/CO/AR→LATAM; "EUROP"→EU; "ARAB"/"AFRICA"/"ASIAN"/"AR/AF"→MENA; else OTHER).
- [ ] Add `testImplementation("junit:junit:4.13.2")` to build.gradle (+ `junit` to version catalog).
- [ ] Create `app/src/test/java/com/itrepos/aiotv/RegionClassifierTest.kt` with real examples:
  "AM | USA SPORTS"→US, "AM | USA NEWS"→US, "24/7 | ENGLISH"→EN, "VIP | FIFA WC26 WEST EUROP"→EU,
  "VIP | FIFA WC26 AR/AF"→MENA, "FIFA-DO| CDN DEPORTES"→LATAM, "UK SPORTS"→UK, ""→OTHER.
- [ ] Run: `./gradlew testDebugUnitTest` → PASS. Commit: `feat(livetv): RegionClassifier + tests`.

### Task 2.2: Tag entities + persist region selection
- [ ] In `LiveTvRepository.refresh`, set `regionTag = RegionClassifier.classify(name)` for channels
  (use the channel's category name; fall back to channel name) and categories.
- [ ] Add `liveRegions: Flow<Set<String>>` + `setLiveRegions(Set<String>)` to `AppDataStore`
  (key `live_regions`, default `{US, UK, EN}`).
- [ ] Build → SUCCESS. Commit: `feat(livetv): region tagging + persisted region selection`.

### Task 2.3: Region filter in ViewModel (temporary chips ok)
- [ ] `LiveTvViewModel`: add `selectedRegions` (from DataStore, default {US,UK,EN}); channels Flow
  filters by `selectedRegions` (+ category); `setRegions(set)` persists. Categories shown are those
  within selectedRegions. Global search still ignores region.
- [ ] Build → SUCCESS.
- [ ] **Author validates on emulator:** default open shows US/UK/EN channels only (far fewer than 27.5k); switching region changes the list; search still finds any channel. Commit: `feat(livetv): region filter (default UK+US+EN)`.
